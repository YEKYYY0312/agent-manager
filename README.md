# Agent DevTools

Agent DevTools is a local-first debugger for AI agent runs. It records an agent execution as a structured trace, then lets developers inspect, analyze, compare, and later replay the run.

[![CI](https://github.com/YEKYYY0312/agent-manager/actions/workflows/ci.yml/badge.svg)](https://github.com/YEKYYY0312/agent-manager/actions/workflows/ci.yml)

Agent DevTools is not a hosted observability platform. The useful wedge is smaller: debug one run locally, compare two runs, replay from a checkpoint, and share a sanitized trace without creating a cloud account.

## What Works Now

- Python SDK for recording runs and steps.
- Decorators and context managers for model calls, tool calls, planner steps, and custom steps.
- Portable `.trace.json` files based on `schemas/trace.schema.json`.
- CLI commands for listing, showing, inspecting, costing, analyzing, diffing, deterministic replay, callable adapter replay, replay comparison, trace experiments, CI regression checks, OTLP JSON export, and OTLP HTTP push.
- CLI commands for privacy-scanning, redacting traces, and importing/searching traces in a local SQLite store.
- Analysis module for cost, latency, failures, retry chains, and simple loop detection.
- SDK adapter contract plus `CallableAgentAdapter` for executing real Python callables under tracing.
- Optional `LangGraphAdapter` for executing compiled LangGraph graphs through `graph.invoke(...)` or node-level `graph.stream(...)` without adding LangGraph as a core dependency.
- Optional `OpenAIAdapter` for tracing Responses API and Chat Completions calls with token usage mapped into step cost, plus opt-in Responses output item expansion.
- Optional `AnthropicAdapter` for tracing Claude Messages API calls with token usage mapped into step cost, opt-in content block expansion, and local tool-use loop execution.
- Adapter replay API for re-running a selected trace step through a real adapter.
- Web UI workbench with Trace Picker, Timeline search/filter, Step Inspector, Analysis, Run Diff, Replay Workbench, Replay Compare, Experiment comparison, Chinese UI text, IndexedDB-backed local imported trace persistence, and browser-side import history.

## Install

For local development from this repository:

```powershell
py -m pip install -e .
agent-devtools --help
```

On Windows, if `agent-devtools` is not found after install, add Python's `Scripts`
directory to `PATH` or run the generated script by full path.

You can also run the CLI directly from source without installing:

```powershell
py packages\cli\agent_devtools_cli\main.py --help
```

Build a local Python package:

```powershell
py -m build
```

Release and publishing notes live in [docs/release.md](docs/release.md).

## Quick Start

From the project root:

```powershell
py examples\simple-agent\demo.py
agent-devtools list traces
```

If you have not installed the editable package, use the source CLI path instead:

```powershell
py packages\cli\agent_devtools_cli\main.py list traces
```

Pick a trace file from the list, then inspect it:

```powershell
agent-devtools show traces\<trace-file>.trace.json --detail
agent-devtools steps traces\<trace-file>.trace.json
agent-devtools cost traces\<trace-file>.trace.json
agent-devtools analyze traces\<trace-file>.trace.json
```

Create a deterministic replay trace from a recorded step:

```powershell
agent-devtools replay traces\<trace-file>.trace.json --start-step <step-id>
```

Run deterministic replay from a Replay Plan JSON with edited tool mocks:

```powershell
agent-devtools replay traces\<trace-file>.trace.json --plan replay-plan.json --output-dir traces
```

Execute an explicit local Python callable from a recorded step:

```powershell
agent-devtools replay-adapter traces\<trace-file>.trace.json --start-step <step-id> --callable path\to\agent.py:run --allow-unsafe-code --output-dir traces
```

Compare an original trace with a replay trace:

```powershell
agent-devtools replay-compare traces\<source>.trace.json traces\<replay>.trace.json
```

Compare two traces:

```powershell
agent-devtools diff traces\<left>.trace.json traces\<right>.trace.json
```

Compare two traces as an A/B experiment:

```powershell
agent-devtools experiment traces\<left>.trace.json traces\<right>.trace.json
```

Fail CI when a candidate trace exceeds regression thresholds:

```powershell
agent-devtools regression-check traces\<baseline>.trace.json traces\<candidate>.trace.json --max-token-delta 100 --max-latency-delta-ms 500 --json
```

Write a sanitized trace copy before sharing:

```powershell
agent-devtools redact traces\<trace-file>.trace.json --output traces\<trace-file>.safe.trace.json
```

Scan a trace for sensitive values without printing those values:

```powershell
agent-devtools privacy-scan traces\<trace-file>.trace.json
```

Import traces into the local SQLite store and search them:

```powershell
agent-devtools store import traces --redact
agent-devtools store list
agent-devtools store search "weather"
```

For a shared or production store, install the optional PostgreSQL extra and pass a database URL explicitly:

```powershell
python -m pip install -e ".[postgres]"
$env:AGENT_DEVTOOLS_DATABASE_URL = "postgresql://agent:change-me@db.example:5432/agent_devtools"
agent-devtools store import traces --redact --database-url $env:AGENT_DEVTOOLS_DATABASE_URL
agent-devtools store list --database-url $env:AGENT_DEVTOOLS_DATABASE_URL
```

Do not commit real `.env` files or database credentials. `.env.example` is only a placeholder template.

Export a trace as OpenTelemetry-compatible OTLP JSON:

```powershell
agent-devtools otel-export traces\<trace-file>.trace.json --redact --output traces\<trace-file>.otlp.json
```

Push a trace directly to an OpenTelemetry Collector OTLP HTTP endpoint:

```powershell
agent-devtools otel-push traces\<trace-file>.trace.json --redact --endpoint http://localhost:4318/v1/traces
```

By default `store import`, `otel-export`, and `otel-push` run a privacy preflight and stop when sensitive values are detected. Use `--redact` to sanitize before writing or pushing, or `--allow-sensitive` only when you explicitly want to continue with the original trace. OTLP export/push also omits step input/output and tool args/results unless `--include-payloads` is provided.

Security defaults: `replay-adapter` refuses to execute local Python callables unless `--allow-unsafe-code` is provided, prints the callable file path and SHA-256 hash for file-based callables, and runs those callables in a child Python process so temporary import path changes do not affect the main CLI process. This is isolation, not a sandbox: only run trusted code. `otel-push` allows HTTPS and loopback HTTP by default; non-loopback HTTP and private/link-local endpoints require explicit `--allow-insecure-endpoint` and/or `--allow-private-endpoint`.

Set this environment variable to make SDK writes redact automatically:

```powershell
$env:AGENT_DEVTOOLS_REDACT_ON_WRITE = "true"
```

Run the Web UI:

```powershell
cd packages\web-ui
npm run dev
```

Open the local URL printed by Vite. The Web UI loads sample traces from `packages/web-ui/public/traces/`, and it can import local `.trace.json` files from the Trace list.
Imported trace contents are encrypted with browser Web Crypto AES-GCM before they are stored in IndexedDB, so refreshes can reopen previously imported traces without keeping trace JSON plaintext in browser storage. localStorage stores only lightweight import metadata.

## Current Status

This repository is a usable MVP for local-first agent debugging. The core trace contract, SDK, CLI, analysis module, Web UI workbench, Chinese UI labels, browser-side imported trace persistence, Replay Workbench plan generation, deterministic replay trace generation with editable tool mocks, callable adapter replay from the CLI, CLI/Web original-vs-replay comparison, adapter-based replay API, callable, LangGraph, OpenAI, and Anthropic runtime adapters, LangGraph node-level streaming traces, opt-in OpenAI Responses output item expansion, opt-in Anthropic content block expansion, Anthropic local tool-use loop execution, CLI/Web experiment comparison, CI regression checks, privacy scanning/redaction, local SQLite trace storage, OTLP JSON export and HTTP push, package metadata, release docs, GitHub templates, and CI workflow are in place. Anthropic streaming expansion, OpenAI Agents/streaming event expansion, and broader framework-specific CLI replay are planned after security audit.
